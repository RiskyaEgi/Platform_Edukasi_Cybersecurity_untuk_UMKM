Software Requirements Specification (SRS)

Nama: Riskya Egi Pratama
NIM: A710230031

1. Pendahuluan

1.1 Tujuan Penulisan Dokumen

Dokumen ini bertujuan untuk mendefinisikan kebutuhan perangkat lunak dari Platform Edukasi Cybersecurity untuk UMKM, mencakup kebutuhan fungsional, antarmuka, non-fungsional, serta proses manajemen proyek berbasis Agile menggunakan JIRA.

1.2 Audien dan Pembaca yang Disarankan

Pengembang perangkat lunak, manajer proyek, penguji (QA), pelaku UMKM, dan penyedia edukasi cybersecurity.

1.3 Batasan Produk

Platform pembelajaran berbasis web yang menyajikan modul edukasi, kuis interaktif, pelaporan kemajuan, forum diskusi, dan sertifikat digital.

1.4 Definisi dan Istilah

  • SRS: Software Requirements Specification
  • UMKM: Usaha Mikro, Kecil, dan Menengah
  • Trainer: Narasumber/pembimbing edukasi
  • Modul: Unit pembelajaran

1.5 Referensi

  • Panduan IEEE 830-1998
  • Dokumentasi REST API
  • Panduan Pengembangan Agile (Scrum)
  • Dokumentasi JIRA Software

2. Deskripsi Keseluruhan

2.1 Deskripsi Produk

Platform ini memberikan edukasi seputar cybersecurity secara mudah, fleksibel, dan terstruktur bagi pelaku UMKM.

2.2 Fungsi Produk

  • Registrasi dan login pengguna
  • Akses modul edukasi cybersecurity
  • Kuis evaluasi dan pelaporan nilai
  • Forum diskusi dengan trainer
  • Penerbitan sertifikat

2.3 Penggolongan Karakteristik Pengguna

PeranTugasHak Akses
UMKMMengakses materi dan mengikuti kuisDasar
TrainerMemberi jawaban di forumTerbatas
AdminKelola modul, user, laporanPenuh

2.4 Lingkungan Operasi

  • Platform: Web-based
  • Browser: Chrome, Firefox, Safari
  • Backend: Django/Node.js
  • Frontend: React
  • Database: PostgreSQL

2.5 Batasan Desain dan Implementasi

  • Bahasa: JavaScript, Python
  • Framework UI: Tailwind CSS
  • Otentikasi: OAuth2 + Email Verification

6. Manajemen Proyek di JIRA

6.1 User Story Mapping

  • Sebagai UMKM, saya ingin belajar agar bisnis aman
  • Sebagai admin, saya ingin mengelola modul
  • Sebagai trainer, saya ingin menjawab pertanyaan

6.2 Sprint Planning (Scrum)

  • Sprint 1: Login, register, peran user
  • Sprint 2: Akses materi, tracking progres
  • Sprint 3: Soal kuis & nilai
  • Sprint 4: Forum diskusi
  • Sprint 5: Sertifikat & laporan
  • Sprint 6: Pengujian & optimasi

6.3 Bug Tracking

Tiket berisi deskripsi bug, langkah reproduksi, prioritas, status, dan assignee.

6.4 Progress Reporting

  • Burndown Chart
  • Dashboard JIRA
  • Daily Stand-up Meeting
